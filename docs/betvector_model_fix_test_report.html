Model Fix Test Report

Manager features caused a Brier regression.
Here's how we fix it.

After adding E40 manager features, the Poisson GLM regressed in 5 of 6 leagues. Root cause: manager_win_pct has strong signal (r=0.20) but at 88% coverage, mean-imputed rows distort the GLM fit. Eight fix variants were tested across all six leagues using walk-forward backtesting.

Key Findings
Finding 01
Ridge regularisation is broken
Produces inf lambdas with the current _add_constant() + predict() pipeline. All Ridge variants produce identical Brier ~0.6666 across all leagues.
BROKEN
Finding 02
Elastic Net works correctly
fit_regularized(alpha=0.05, L1_wt=0.5) constrains coefficients properly. Wins 4 out of 6 leagues. Average Brier 0.5943 vs current 0.5983.
WORKS
Finding 03
E39 lineup features help
squad_rotation_index and formation_changed improve EPL accuracy by −0.0025 Brier. These features have strong coverage and genuine predictive signal.
HELPS
Finding 04
E40 manager features hurt
manager_win_pct overfits in the unregularised GLM. At 88% coverage, the 12% mean-imputed rows create artificial signal the model latches onto. EPL regression: +0.0185 Brier.
HURTS
